# FluxMesh 流量面 MVP 设计草案（简版）

版本：v0.6（2026-03-18）

## 1. 背景

FluxMesh 控制面已具备：

- 节点注册与拓扑管理
- 服务配置管理（`/mesh/services/`）
- 软状态同步（SoftKV + Gossip）

下一阶段目标是实现可运行的流量面，让每个节点具备本地转发与自治调度能力。

## 2. 目标与非目标

目标：

- 支持 L7 HTTP 代理（MVP 仅支持 HTTP）
- 支持按 Host/Path 的虚拟主机路由（类似 Nginx 的 `server_name + location`）
- 支持按服务配置进行路由与转发
- 支持基于实时负载的自适应选路
- 支持跨节点中继（Relay）以适配跨地域/跨网络隔离场景
- 以高性能与轻量化为首要工程方向
- 参考 Caddy 的工程理念（声明式、可组合、默认可用、热更新友好），但保持 FluxMesh 自身配置模型
- 产品定位为“分布式版 Nginx”，强调多节点自治与一致配置下发

非目标（MVP 暂不实现）：

- iptables/eBPF 流量劫持
- 完整 xDS/Envoy 生态对接
- 复杂多协议特性（TCP/gRPC 后续扩展）

## 3. 核心能力清单（MVP）

### 3.1 本地代理与路由

- 每个节点提供本地流量入口（HTTP）
- 按服务 `listener.addr + listener.port` 声明入口；同地址同端口由运行时自动复用监听
- 按 Host + Path 进行路由匹配
- 按服务名 + 路径前缀匹配路由规则
- 根据权重和节点运行态选择目标实例

### 3.2 自适应负载均衡

- 基础策略：加权轮询
- 增强策略：结合 RTT、CPU、内存、错误率进行打分
- 默认策略：负载优先（load-first）
- 支持通过配置切换策略（例如 latency-first / load-first）
- 失败实例短时降权，避免持续放大故障

### 3.3 健康检查与失败恢复

- 主动探活（定时）
- 被动熔断（连续失败阈值）
- 半开恢复机制（小流量试探）

### 3.4 Mesh 中继（核心亮点）

- 直连优先，失败后自动选择中继节点
- 请求携带跳数上限（TTL）防环路，默认最大跳数为 2
- 最大跳数支持全局配置 + 服务级覆盖
- 中继失败自动切换候选路径

### 3.5 可观测性

- 请求级指标：成功率、延迟、重试次数
- 路由级指标：目标节点命中率、失败原因
- 中继指标：中继命中率、额外时延

## 4. 配置与状态分层

强一致（etcd）：

- 服务路由规则
- 权重与治理阈值（超时、重试、熔断）
- 中继最大跳数（全局与服务级）
- 调度策略选择（全局默认与服务级覆盖）

最终一致（SoftKV）：

- 节点负载快照（CPU/内存/load）
- 节点间链路质量（延迟、可达性）

本地瞬时状态（内存）：

- 连接池、in-flight 计数
- 熔断器状态
- 近期 EWMA 延迟与错误率

## 5. 请求处理流程（简化）

1. 接收请求，匹配服务与路由规则。
2. 计算候选目标列表（可直连 + 可中继）。
3. 依据负载评分选择最优路径。
4. 发起转发；失败则按预算重试或切中继。
5. 记录指标并更新本地短周期统计。

## 6. 配置模型与优先级

配置优先级（高到低）：

1. 服务级配置
2. 全局默认配置
3. 系统内置默认值

MVP 建议默认值：

- 调度策略：load-first
- 中继最大跳数：2
- 重试预算：按服务级配置

说明：

- 若服务未显式配置，则继承全局默认。
- 若全局也未配置，则使用系统内置默认值。

### 6.1 极简字段（MVP 必需）

先只保留实现必需字段，其余参数延后到下一版。

全局字段：

| 字段 | 数据类型 | 可选/必选 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `traffic.lb.strategy` | `str` | 可选 | `load-first` | 支持 `load-first`、`latency-first`、`round-robin`、`random`。 |
| `traffic.relay.max_hops` | `int` | 可选 | `2` | 中继最大跳数。 |
| `traffic.retry.max_attempts` | `int` | 可选 | `1` | 全局重试尝试次数默认值；`1` 表示默认不重试。 |
| `traffic.retry.budget_ratio` | `float` | 可选 | `0` | 全局重试预算比例，`0` 表示默认不启用重试预算。 |

服务级字段（`traffic_policy`）：

| 字段 | 数据类型 | 可选/必选 | 默认行为 | 说明 |
| --- | --- | --- | --- | --- |
| `proxy.layer` | `str` | 可选 | `l7-http` | 预留层级能力：`l7-http`（MVP）/ `l4-tcp`（后续）。 |
| `protocols` | `list[str]` | 可选 | `["http"]` | 预留协议字段，MVP 固定为 HTTP。 |
| `listener.addr` | `str` | 可选 | `0.0.0.0` | 服务监听地址；可指定 `127.0.0.1`、`0.0.0.0` 或具体网卡 IP。 |
| `listener.port` | `int` | 必选 | 无 | 服务声明监听端口；运行时可自动与其他服务复用同地址同端口监听。 |
| `lb.strategy` | `str` | 可选 | 继承全局 | 可按服务切换调度策略（`load-first` / `latency-first` / `round-robin` / `random`）。 |
| `retry.max_attempts` | `int` | 可选 | 继承全局 | 建议不超过 `2`。 |
| `retry.budget_ratio` | `float` | 可选 | 继承全局 | 服务级重试预算。 |
| `relay.max_hops` | `int` | 可选 | 继承全局 | 服务级中继跳数覆盖。 |

路由字段（`routes[]`）：

| 字段 | 数据类型 | 可选/必选 | 默认行为 | 说明 |
| --- | --- | --- | --- | --- |
| `hosts` | `list[str]` | 可选 | `*` | Host 匹配列表，例如 `api.example.com`。 |
| `path_prefix` | `str` | 必选 | 无 | Path 前缀匹配。 | 
| `destination` | `str` | 必选 | 无 | 目标后端组名或服务名。 |
| `weight` | `int` | 可选 | `100` | 同级路由权重。 |

最小校验规则：

- 服务级 > 全局 > 系统默认。
- `listener.addr` 未配置时按 `0.0.0.0` 处理。
- `listener.addr` 必须是合法监听地址（`0.0.0.0`、`127.0.0.1` 或本机可绑定 IP）。
- `listener.port` 必须是合法端口（`1` 到 `65535`）。
- 多个服务声明相同 `listener.addr + listener.port` 时，运行时必须复用同一个监听 socket。
- 多个服务共享同地址同端口时，必须可通过 `hosts` 或 `path_prefix` 稳定区分路由。
- `relay.max_hops` 取值建议 `1` 到 `4`，默认 `2`。
- `routes[].hosts` 未配置时按 `*` 处理。
- `proxy.layer=l7-http` 时需要 `routes[]`；`l4-tcp` 预留为后续扩展。
- `routes[].destination` 必须能解析到已存在的后端组或服务。

### 6.2 后端组定义（backend_groups）

为避免把地址列表直接写进路由，MVP 使用“后端组”承载真实上游节点。

后端组字段：

| 字段 | 数据类型 | 可选/必选 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `name` | `str` | 必选 | 无 | 后端组唯一名称，供 `routes[].destination` 引用。 |
| `targets` | `list[dict]` | 必选 | 无 | 后端实例列表。 |
| `targets[].addr` | `str` | 必选 | 无 | 上游地址，如 `10.0.0.12:8080`。 |
| `targets[].weight` | `int` | 可选 | `100` | 实例权重。 |
| `targets[].tags` | `dict` | 可选 | `{}` | 可选标签，用于后续扩展（机房、版本等）。 |

最小校验规则：

- `backend_groups[].name` 全局唯一。
- 每个后端组 `targets` 至少包含一个可用地址。
- `routes[].destination` 必须命中后端组或服务，否则拒绝加载。

### 6.3 配置示例

全局：

```json
{
  "traffic": {
    "lb": {
      "strategy": "load-first"
    },
    "retry": {
      "max_attempts": 1,
      "budget_ratio": 0
    },
    "relay": {
      "max_hops": 2
    }
  }
}
```

服务级：

```json
{
  "name": "payment-svc",
  "routes": [
    {
      "hosts": ["pay.example.com"],
      "path_prefix": "/",
      "destination": "payment-v1",
      "weight": 100
    }
  ],
  "traffic_policy": {
    "proxy": {
      "layer": "l7-http"
    },
    "listener": {
      "addr": "0.0.0.0",
      "port": 18080
    },
    "retry": {
      "max_attempts": 2,
      "budget_ratio": 0.15
    },
    "relay": {
      "max_hops": 3
    }
  }
}
```

后端组：

```json
{
  "backend_groups": [
    {
      "name": "payment-v1",
      "targets": [
        {
          "addr": "10.0.1.10:8080",
          "weight": 100
        },
        {
          "addr": "10.0.1.11:8080",
          "weight": 100
        }
      ]
    }
  ]
}
```

### 6.4 设计取向

- 声明式：配置描述目标状态，不要求重复声明细节参数。
- 默认可用：默认值即可跑通，进阶能力按需开启。
- 可组合：监听、路由、重试、中继按模块组织。
- 热更新友好：优先支持无重启生效，非法配置原子拒绝，支持使用api动态添加修改删除配置。
- 路由语义贴近 Nginx：优先 Host，再做 Path 前缀匹配。
- 监听语义贴近 Nginx/Caddy：服务按 `listener.addr + listener.port` 声明，运行时自动复用同地址同端口监听。
- 协议层能力在服务级声明：为后续 L4/L7 共存预留扩展位。

## 7. 三阶段落地计划

阶段 1：HTTP 单跳可用

- 完成路由匹配、加权转发、基础超时与重试
- 完成最小可观测指标

阶段 2：自适应调度

- 接入 SoftKV 运行态指标
- 增加降权/熔断/半开恢复

阶段 3：跨域中继

- 增加中继候选选择与防环路
- 完成中继路径指标与稳定性验证

## 8. 已确认决策

- 自适应调度默认策略采用负载优先；并提供可配置策略切换。
- 中继最大跳数默认值为 2；支持全局与服务级配置。
- 监听入口由服务级 `listener.addr + listener.port` 声明；同地址同端口服务由运行时自动复用监听。
- 不再引入全局 traffic.mode。
- 支持按 Host/Path 路由，整体能力目标对齐“分布式版 Nginx”。
- 服务级预留 L4/L7 代理层级字段，MVP 仅启用 l7-http。
- 重试预算按服务级配置。
- MVP 阶段仅支持 HTTP；gRPC 放到下一迭代。
- 性能与轻量化作为实现和取舍的首要方向。

## 9. 性能与轻量化原则

- 先保证请求路径短链路，避免引入重依赖与重代理层。
- 优先减少内存分配与对象复制，控制每请求额外开销。
- 可观测性以必要指标为主，避免高基数标签导致系统自我放大。
- 重试与中继必须受预算约束，防止故障时放大流量。
- 默认配置以保守稳定为主，复杂策略通过配置渐进开启。
- 配置变更优先热更新路径，减少重启和连接抖动。

## 10. 验收指标（建议）

- 故障恢复：单节点故障后恢复到稳定成功率的时间
- 性能：P95 延迟增幅与吞吐变化
- 稳定性：重试放大倍数与错误率
- 资源：每节点 CPU/内存开销
